# x86_64 状态

## 当前建议

如果目标是尽快熟悉仓库，**不建议**把 `x86_64` 作为第一条验证路径。

当前更稳妥的默认体验：

- ArceOS / StarryOS：优先使用 **riscv64**
- Axvisor：优先使用 **aarch64**

## 原因

- 现有开发说明与示例主要围绕 riscv64 和 aarch64 编写
- Axvisor 的 `x86_64` 路径在当前仓库中更多是代码和配置占位
- 统一测试和文档中的高频示例不以 `x86_64` 为主

## 何时参考本文档

- 正在做 `x86_64` 平台适配
- 需要核对某个 crate 的跨架构条件编译行为
- 准备补齐或改造 `x86_64` 文档与验证链路

相关文档：[Arch / Target 映射](../../design/build/arch)
